# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A mobile-first PWA (Italian) to study for the exam of "Corso di Formazione per Istruttori di 1° Livello (Padel) — FITP, A.A. 2026". Static site, no build step, no package manager, no framework — plain HTML/CSS/JS loaded directly by the browser. All quiz/study content is transcribed from `Corso IS1 2026 Ultimo.pdf` (git-ignored, ~18MB source slide deck, not needed to run the site).

## Running locally

The service worker requires a real origin (won't register from `file://`):

```bash
python3 -m http.server 8080
# then open http://localhost:8080
```

There is no build, lint, or test tooling in this repo (no `package.json`). Changes to `index.html`, `style.css`, `data.js`, `script.js` are live on reload — just refresh the browser. When editing cached assets, bump the `CACHE` version string in `service-worker.js` (currently `is1-2026-v1`) or the browser will keep serving stale files from the old cache.

## Architecture

Four files carry the whole app, loaded in this order by `index.html`: `data.js` → `script.js`, with Three.js and GSAP pulled from CDN.

- **`data.js`** — pure content, no logic. Three top-level arrays:
  - `SECTIONS`: 24 sections, each `{id, name, icon, cat, qa:[{q, a, e, wrong:[...]}]}` — `cat` is one of `sistema | deontologia | campo | didattica | tattica | tecnica`. This one array feeds Sezioni, Flashcard, and Quiz.
  - `NUMBERS_DATA`: key numbers to memorize, `{val, lbl, ctx}`.
  - `PLAN_DAYS`: the 7-day study plan shown on Home.

- **`script.js`** — everything else, organized in `// ====` banner sections in this order: Three.js background scene → vibration helpers → particle/celebration FX → screen transition animations (GSAP) → **state (localStorage)** → **navigation** → **screen renderers** → runtime app-icon generation (canvas) → init/bootstrap. Read top to bottom; later sections depend on earlier ones (e.g. renderers call the FX/vibration helpers).

- **Single-page, no router**: `index.html` has one `<div class="screen">` per screen (`home`, `sezioni`, `detail`, `flash`, `quiz`, `numeri`), all present in the DOM, shown/hidden via `goTo(name, btn)` in `script.js`. Each screen's HTML is generated by its own `render*()` function (`renderHome`, `renderSezioni`, `renderDetail`, `renderFlash`, `renderQuizStart`/`renderQuizQ`/`renderQuizResult`, `renderNumeri`) which builds an HTML string and sets `innerHTML` — there is no virtual DOM or component system. Interactive elements call back into global functions via inline `onclick="..."` handlers.

- **State/persistence**: everything is in one localStorage key, `LS = 'is1_2026_v1'`, loaded into the in-memory `store` object via `load()`/`save()`. This holds: theme (`store.theme`), per-question read/known flags keyed `r_<sectionId>`, per-day plan completion keyed `day_<n>`, and `store.quizHistory` (array of past quiz results). There is no backend — all progress is local to the device/browser.

- **Three.js background**: an IIFE at the top of `script.js` (`initThree`) draws an animated wireframe padel court + ball arc on the fixed `#three-bg` canvas, independent of the rest of the app; it no-ops if `THREE` isn't loaded or the canvas is missing.

- **PWA/offline**: `service-worker.js` precaches the app shell (`ASSETS` list) on install and does cache-first for same-origin requests, network-first-with-cache-fallback for cross-origin (CDN/font) requests. `manifest.json` + `icons/` provide install metadata; `script.js` also generates a data-URL favicon/apple-touch-icon at runtime via canvas instead of using a static file for those.

## Adding study content

New sections/questions/numbers go in `data.js` only — no schema/build validation exists, so keep new entries shaped exactly like their neighbors (same keys, `wrong` arrays with distractor strings for quiz answers). The Quiz screen pulls `wrong` answers as distractors and category filters (`setQuizCat`) rely on the `cat` field matching one of the six existing category slugs.
